SrvrPowerCtrl plugin setup instructions for unsupported operating systems

SrvrPowerCtrl's job is to help you control power states (sleeping / waking / etc.) on your SqueezeCenter server hardware. To do that, SrvrPowerCtrl needs to be able use some “helper” applications on your server. These applications differ according to which operating system you are using on the SqueezeCenter server. But whatever your server setup, you'll need to do a certain amount of manual tweaking of those helper applications and SrvrPowerCtrl's settings in order to get things working as they ought. Hopefully, these instructions will help you get everything set “just right.”



WARNING: your server's operating system, which has been identified as “[% distro %]”, may not be supported by SrvrPowerCtrl ‘out of the box’. That said, you may be able to get SrvrPowerCtrl to work with [% distro %]. Your first step will be to determine which system commands supported by [% distro %] are used to execute the following functions:

  • System shutdown
  • System restart
  • System suspend
  • System hibernate

    • Next, you’ll need to determine which user account SqueezeCenter/Squeezebox Server runs under on your operating system.

    Next, you’ll need to determine how to give that user account permissions to execute those system commands without being prompted for a password. For unix and linux based OSs, this is usually accomplished through modifications to the /etc/sudoers file.

    Finally, you’ll need to fix up the command lines on the SrvrPowerCtrl settings page to point to the appropriate system commands. For many operating systems, you’ll need to preface the commands with a “sudo” or like command so that the system command may be run with the appropriate privileges.

    Consult the readme.txt file for details about enabling SrvrPowerCtrl for Windows, RedHat, Debian or MacOSX...whichever one most closely matches your operating system, “[% distro %]”.
